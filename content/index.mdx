# Web3 Dev Lab

> 一个通过动手实践来学习Web3开发的项目集

## 这是什么

这是我在学习Web3开发过程中的实践记录和总结。我发现单纯看文档和教程很难真正理解区块链开发的核心概念，只有通过完整地做出一个项目，才能把知识点串联起来。

这个项目是开源的，分享给所有想学习Web3开发的同学。希望我们能一起进步。

## 为什么做这个

学习Web3开发有很多挑战：
- **概念多而碎片化**：密钥、签名、Gas、EVM、Solidity、ABI...每个都要懂
- **缺少完整示例**：大多数教程只讲单个知识点，缺少端到端的实践
- **不知道如何上手**：看完文档不知道该写什么，怎么验证

所以我决定通过**5个递进式的项目**来学习：

1. **多链钱包** - 从最基础的密钥管理开始，理解签名和交易
2. **AMM DEX** - 学习DeFi的核心：流动性池和自动做市
3. **借贷协议** - 深入风控模型、清算机制和预言机
4. **跨链桥** - 理解跨链通信和资产桥接
5. **ZK电路** - 探索零知识证明和隐私保护

每个项目都会从零开始搭建，包含完整的代码实现（Go + Solidity）和详细的讲解文档。

## 每个项目包含什么

- **架构设计** - 系统是如何组织的，各个模块的职责
- **核心概念** - 拆解关键知识点，配合图解说明
- **分步教程** - 从环境搭建到完整运行的每一步
- **代码示例** - 可以直接运行的完整代码（Go + Solidity）
- **测试网演示** - 在测试网上实际部署和交互

## 对社区的理解

我相信好的学习资源应该具备这些特点：

**1. 完整而非碎片**  
一个完整的项目能让你理解整个系统是如何工作的，而不是只知道某个API怎么调用。

**2. 实践而非理论**  
光看文档永远不够，必须自己动手写代码、部署合约、调试问题，才能真正掌握。

**3. 开放而非封闭**  
所有代码和文档都开源，任何人都可以使用、修改、提出改进建议。

**4. 共同成长**  
我也在学习过程中，文档可能有错误或不完善的地方，欢迎指出。我们一起讨论、一起进步。

## 适合谁

- **有编程基础的开发者** - 你不需要是区块链专家，但应该熟悉至少一门编程语言
- **想系统学习Web3的同学** - 不满足于看完教程就忘，想通过项目深入理解
- **希望动手实践的学习者** - 不怕遇到问题，愿意调试和探索
